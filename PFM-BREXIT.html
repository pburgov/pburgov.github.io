1 Introducción ↑ Menú¶

Para la realización del Proyecto Fin de Máster, he escogido la opción relativa a los resultados del Brexit. Esta elección se justifica en que desde el anuncio por parte de James Cameron en 2013 de la realización del Referendum, la eventual salida del Reino Unido de la UE me ha provocado un gran interés por 2 motivos:

  • En primer lugar y fundamentalmente por la posible resaca, que derivada de tal decisión, pudiera sufrir la UE y por tanto España.
  • Porque llama poderosamente la atención la tendencia de UK al aislamiento, precisamente en el mundo con tintes globales en el que nos encontramos, y en el que, metafóricamente hablando, un gran número de fronteras se están diluyendo.

1.1 Objetivo del estudio ↑ Menú¶

El objetivo principal es el de buscar posibles causas o relaciones comunes, existentes entre los distintos distritos electorales, que hayan decantado el resultado final hacia la salida de la UE.

1.2 Datasets de trabajo ↑ Menú¶

1.2.1 Descripción ↑ Menú¶

El dataset principal es uno de los 4 propuestos por el claustro del Máster, y que se encuentra disponible en la web kaggle bajo el epígrafe 2016 EU Referendum in the United Kingdom . Dicho dataset consta de 2 archivos con formato csv:

  • census.csv: En el se recogen los datos censales de los distintos distritos de Inglaterra, Gales, Escocia e Irlanda del Norte. Los primeros 44 registros del archivo muestran los datos agregados primero por país, luego por región y finalmente por condado. Los 380 registros restantes corresponden a los datos desglosados por distrito.

  • referendum.csv: En este fichero se recogen los datos principales de las votaciones realizadas en cada uno de los distritos tales como, asistencia, votos esperados, votos válidos y rechazados, valores y porcentajes resultantes del escrutinio, etc.

Como complementación de este dataset, a partir de los datos proporcionados por la web oficial de UK Nomis - Official Labour Market Statistics he creado un nuevo archivo también con formato csv denominado social.csv. Dicho archivo contiene los siguientes campos:

  • Area_Code: Código identificativo del distrito. (String)
  • jobseeker: Ratio de gente por distrito en búsqueda de empleo. (Double)
  • state_pension:Ratio de gente que recibe una pensión básica del estado para prevenir la pobreza tras la jubilación. (Double)
  • income: Ratio de gente por distrito que recibe una paga suplementaria para llegar a fin de mes. (Double)
  • ESA: Ratio de gente que recibe una prestación por desempleo, (Double)
  • coast: Indica que el distrito está en la costa. Esta variable parte de la premisa tradicional, quizás errónea, de que los pueblos costeros por su mayor contacto con otras culturas son más abiertos que los pueblos de interior. Éstos por su mayor aislamiento geográfico, se encuentran más encerrados en sí mismos, y por tanto podría contribuir a que el voto de la gente se decantase por el Brexit (Integer Valores 1/0)

1.2.2 Aclaración ambito estudio. Inglaterra, Gales y Escocia ↑ Menú¶

El dataset referendum no recoge los datos segregados por distrito de Irlanda del Norte. De la misma manera, el dataset census no contiene los datos de Gibraltar, por ello se prescinde de ambos, limitándose el estudio a Inglaterra, Gales y Escocia

import re
def remove_commas(line):
    """
      Función que devuelve una cadena eliminando las
      comas que puedan existir entre 2 comillas
      :param line: línea del RDD
      :return string: cadena parseada
    """
    return re.sub(r'(?!(([^"]*"){2})*[^"]*$),', '', line)

# Método para sustituir los valores nulos por 0
def check_null(value):
    if len(str(value)) != 0:
        return int(value)
    return 0

# Método para categorizar el tamaño de los distritos
def get_district_size(population):
    value = 7
    if population < 50000:
        value = 1
    elif 50000 <= population < 100000:
        value = 2
    elif 100000 <= population < 200000:
        value = 3
    elif 200000 <= population < 300000:
        value = 4
    elif 300000 <= population < 400000:
        value = 5
    elif 400000 <= population < 500000:
        value = 6
    return int(value)  
    
    
# Método para realizar las transformaciones en el RDD census_rdd
def transform_census_rdd(line):

    elements = []
    elements.append(str(line[1]))
    elements.append(str(line[2]))
    elements.append(get_district_size(int(line[3])))
    
    # Calculamos los totales de posibles votantes agrupados en 3 tramos
    age_20to40 = int(line[8]) + int(line[9]) + int(line[10]) + int(line[11])
    age_40to65 = int(line[12]) + int(line[13]) + int(line[14]) + int(line[15]) + \
                 int(line[16])   
    age_65plus = int(line[17]) + int(line[18]) + int(line[19]) + int(line[20]) + \
                 check_null(line[21]) + check_null(line[22])
    
    # Calculamos el total de población que puede votar
    total = age_20to40 + age_40to65 + age_65plus 
    
    # Y los porcentajes de cada uno de los grupos
    per_20to40 = (age_20to40 / float(total)) * 100
    per_40to65 = (age_40to65 / float(total)) * 100
    per_65plus = (age_65plus / float(total)) * 100
    
    # Calculamos el total de población de 0-15 años
    age_0to15 = int(line[4]) + int(line[5]) + int(line[6])
    
    # Índice de envejecimiento de la población.
    indice_envejecimiento = (age_65plus/float(age_0to15)) * 100
    
    elements.append(round(per_20to40, 2))
    elements.append(round(per_40to65, 2))
    elements.append(round(per_65plus, 2))
    elements.append(round(indice_envejecimiento, 2))

    return elements

# Método para realizar las transformaciones en el RDD referendum_rdd
def transform_referendum_rdd(line):

    elements = []
    elements.append(str(line[3]))
    elements.append(str(line[2]))
    elements.append(float(line[8]))
    elements.append(float(line[20]))
    elements.append(float(line[19]))
    result = 0.0
    if int(line[12]) > int(line[11]):
        result = 1.0
    elements.append(result)
    
    return elements

De la observación de la matriz de correlación podemos tomar diversas ideas a tener en cuenta posteriormente:

  • A excepción de leave, que representa el porcentaje de votos que optan por la salida de la UE, no hay ninguna variable altamente relacionada con la variable result. Esta relación es lógica ya que result es función de leave. No vamos, por tanto, a prescindir de momento de ninguna de ellas por este motivo.
  • Con la edad de los votantes aumenta también la tendencia a votar al Brexit.
  • El voto a la salida disminuye con el aumento de la población del distrito.

  • Resulta curioso como la variable jobseeker, que representa el ratio de gente en búsqueda de empleo esté tan poco relacionada con el resultado, sobre todo en un proceso como el Brexit, en el que se ha puesto sobre la mesa continuamente el argumento de la inmigracion y de la pérdida de trabajo de los locales en favor de los extranjeros.

    Relacionada con esta variable se encuentra ESA que es el ratio de gente que percibe una ayuda estatal para la búsqueda de empleo. Al igual que jobseeker está muy poco relacionada con el voto. Es de hecho la 2ª menos relacionada.

  • income que representa el ratio de gente que recibe una ayuda estatal complementaria para poder llegar a fin de mes, guarda una cierta relación con el resultado; aunque no mucha.

    Con la que sí presenta una elevada correlación es con la variable ESA. Se podría intuir que ambas ayudas son comunes a una misma fracción de la población.

  • pension es el ratio de gente que recibe una pensión estatal para prevenir la pobreza tras la jubilación. Este valor es prácticamente el mismo que el tramo de población 65plus y en ambos casos suponen las correlaciones más elevadas con respecto a la salida de la UE.

    A priori este valor podría sustentarse simplemente en el factor edad, ya que como hemos visto, la gente mayor ha acudido a votar masivamente.

  • Atendiendo a la variable coast la cercanía al mar favorece el voto a la permanencia. Más adelante podremos comprobarlo con más detalle.

  • La variable rejected representa el porcentaje de votos no válidos. Aunque se ve una gran correlación (-0.314) comparada con el resto de las variables, vamos a prescindir de ella por dos razones

    • rejected aglutina a los votos en blanco, sobres sin papeleta, papeletas no oficiales, múltiples marcas, y papeletas escritas, con tachaduras o enmiendas. Todo esto supone 24.958 votos frente al total de 32.766.647 de votos emitidos; un 0.076%.

    • Si quisieramos tener cierta certeza de qué votos se han emitido de esta manera conscientemente por el elector, creo que deberíamos tener en cuenta únicamente los votos que no han sido marcados (en blanco) y la ausencia de papeleta en el sobre. Considerando esto, tendríamos únicamente 14.981 votos en estas circunstancias; que suponen tan solo un 0.046%.

def set_style_to_axes(ax, **kwargs):   
    """
    Función que nos permite por un lado reutilizar código, 
    y por otro personalizar los gráficos de manera que sigan un 
    patrón estético.
    :param ax: gráfico a personalizar
    """
    ancho_barra = kwargs.get('ancho_barra', 0.4)
    value_size = kwargs.get('value_size', 13)       
    ylabel_text = kwargs.get('ylabel_text', None)       
    xlabel_text = kwargs.get('xlabel_text', None)       
    xticklabels = kwargs.get('xticklabels', None) 
    ax_title = kwargs.get('ax_title', None) 
    
    
    for p in ax.patches:
        ancho_actual = p.get_width()
        diferencia = ancho_actual - ancho_barra
        p.set_width(ancho_barra)
        p.set_x(p.get_x() + diferencia*0.5)
        # Escribimos los valores en las barras
        # centrándolos
        if p.get_height() > 0:
            text_pos = -20
            color='white'
            if  p.get_height() < 10:
                text_pos = 10
                color = '#282828'
            ax.annotate(int(p.get_height()),
                        (p.get_x() + p.get_width()/2.0,
                         p.get_height()),
                         ha='center', va='center', fontsize=value_size,
                         color=color, fontweight='bold', xytext=(0, text_pos),
                         textcoords='offset points')

    ax.set_ylabel(ylabel_text)
    ax.set_xlabel(xlabel_text)        
    
    ax.yaxis.set_label_coords(-0.04, 0.5)
    ax.xaxis.set_label_coords(0.5, -0.1)
    
    if ax_title is not None:
        ax.set_title(ax_title)   
    
    if xticklabels is not None: 
        ax.set_xticklabels(xticklabels)
    
    for label in ([ax.xaxis.label, ax.yaxis.label] + 
                  ax.get_xticklabels() + ax.get_yticklabels()):
        label.set_fontsize(10)
        label.set_color(palette[1])
        label.set_weight('bold')

4.2.4 Those were the days ↑ Menú¶

Una de las consecuencias inmediatas que se ha producido tras conocerse los resultados del Brexit, es la aparición de una brecha generacional en el Reino Unido, motivada por la sensación por gran parte de la población más joven, de que los mayores de su país les han robado el futuro. Numerosos medios británicos de relevancia como la BBC, The Telegraph, entre otros, se han hecho eco del profundo malestar existente.

La conjunción de astros que dieron lugar al conocido desenlace, no es otra que la añoranza de los ancianos británicos por un pasado que recuerdan más esplendoroso e imperial; siendo esa nostalgia por lo que ellos llaman a rose-tinted past, lo que les movilizó masivamente a las urnas. Esto unido a la falta de motivación de los más jóvenes por ir a votar: la población estimada entre los 20 y 24 años en el 2015 era de 3.806.471 , de los cuales únicamente 492.306 se habían registrado para votar en los meses previos al referéndum. Teniendo en cuenta que la opción de la salida ganó por 1.269.501 votos, las cosas podrían haber tomado un cariz muy distinto.

Volviendo a la matriz de correlación, observamos que la variables relativas a la edad de los votantes presentan las siguientes correlaciones con la variable result:

      40to65 =  0.257
      65plus =  0.327

De lo que a priori se deduce, que el aumento del porcentaje de votantes de mayor edad inclina la balanza a favor del brexit.

A la hora de tratar los datos relativos a la edad, me he encontrado con que...

El dataset referendum no contiene el desglose de los votantes por edad, y en la web oficial tampoco se proporciona dicho detalle.

Así pues, la única forma de contrastar resultados con la edad de los votantes es usar los datos del dataset census. Éste contiene los valores poblacionales de cada uno de los distritos agrupados en tramos de 5 años.

En UK la edad mínima legal para votar es 18 años; sin embargo parte de la población que debería incluir en el estudio como posibles votantes, se encuentra en el dataset dentro del campo [Age 15 to 19].

Como no puedo realizar esa distinción he decidido prescindir de los posibles votantes de 18 y 19 años, y realizar el estudio con los mayores de 20 años. Justifico esta decisión en base a 3 puntos:

  • La imposibilidad de desglose comentada.
  • Aun cuando pudiese tener los datos de la población de 18 y 19 años, esto tampoco garantiza que sean votantes finales.
  • La idea básica del análisis es confrontar la distribución de edad de los distritos con el resultado electoral, sean o no votantes.

Así pues, he agrupado la población en 3 tramos de edad, de 20 a 40 años, de 40 a 65, y de 65 en adelante. En esta última se reunen los más que probables jubilados (en UK ya NO existe una edad de jubilación forzosa a los 65), y por tanto los más susceptibles de ansiar una vuelta a tiempos pretéritos. Estos 3 tramos representan básicamente 3 generaciones distintas de posibles votantes.

5.1 Adecuación de Variables Categóricas, Ensamblado y Flujo de Trabajo ↑ Menú¶

Antes de comenzar con la aplicación de los modelos, es necesario someter al dataframe a una serie de transformaciones para que puedan ser usados por aquellos.

Para ello nos valemos de un flujo de trabajo o Pipeline, que encadenará los siguientes trabajos:

  • Transformación de las variables categóricas de tipo numérico mediante la función OneHotEncoder.

  • Adición del resto de variables numéricas del dataframe.

  • Ensamblaje de todas las variables predictoras en un vector de tipo numérico haciendo uso de la función VectorAssembler.

Todos estos pasos, así como la propia aplicación del Pipeline al dataframe se realizará con la función get_transformed_dataframe, definida para tal efecto y que nos permite reutilizar código.

Esta función, que nos devuelve el dataframe transformado y en condiciones de ser usado con posterioridad, consta de 3 parámetros:

  • model_df: dataframe a transformar
  • catCols: variables categóricas de tipo numérico
  • numCols: restos de variables numéricas NO categóricas

Aclaración parámetro catCols:

Las únicas variables categóricas de tipo numérico son coast y size. A pesar de ser usadas ambas posteriormente tanto en los modelos NO supervisados como en los supervisados, se pasan como parámetro y no se incluyen directamente en el cuerpo de la función get_transformed_dataframe por razones formales.

from pyspark.ml.feature import OneHotEncoder, VectorAssembler
from pyspark.ml import Pipeline


def get_transformed_dataframe(model_df, catCols, numCols):
    """
    Función que nos devuelve el dataframe que se pasa como parámetro
    transformado y listo para ser usado por los modelos posteriores
    :param model_df: dataframe que va a ser transformado
    :param catCols: variables categóricas de tipo numérico
    :param numCols: variables numéricas no categóricas.
    :return df_transformed: dataframe transformado
    """
    # stages en el Pipeline
    stages = [] 
    
    for catCol in catCols:
        # Con OneHotEncoder convertimos variables categóricas en Vectores Dispersos 
        encoder = OneHotEncoder(inputCol=catCol, outputCol=catCol+"Dummy")
        # Añadimos el encoder a stages
        stages += [encoder]

    inputCols = map(lambda c: c + "Dummy", catCols) + numCols
    
    # Con el transformador VectorAssembler obtenemos la columna de salida
    # features
    assembler = VectorAssembler(inputCols=inputCols, outputCol="features") 
    
    # Añadimos el assembler a stages
    stages += [assembler]
    
    # Definimos el pipeline con las operaciones en orden
    pipeline = Pipeline(stages=stages)

    # Aplicamos las transformaciones al dataset
    pipelineModel = pipeline.fit(model_df)
    
    # Transformamos el dataframe original
    df_transformed = pipelineModel.transform(model_df)

    # Antes de devolver el dataframe transformado, le 
    # quitamos las columnas dummy.
    for catCol in catCols:
        df_transformed = df_transformed.drop(catCol+"Dummy")

    return df_transformed

from pyspark.ml.evaluation import BinaryClassificationEvaluator
from pyspark.ml.tuning import CrossValidator

# Declaramos globalmente el BinaryClassificationEvaluator    
evaluator = BinaryClassificationEvaluator(labelCol='result')


def get_predictions(estimator, paramGrid):
    """
    Función para aplicar una Validación Cruzada a una 
    clasificación con BinaryClassificationEvaluator a distintos
    modelos
    :param estimator: modelo al que sometemos a CV
    :paramGrid: parámetros usados por el estimador
    :return predictions, modelo
    """
    # Declaramos el CrossValidator
    cv = CrossValidator(estimator=estimator, 
                        estimatorParamMaps=paramGrid, 
                        evaluator=evaluator, 
                        numFolds=5)
    
    # Creamos el modelo aplicando el algoritmo al conjunto de entrenamiento
    modelo = cv.fit(train)
    
    # Aplicamos el modelo sobre el conjunto de test para obtener las predicciones
    predictions = modelo.transform(test)
    
    return predictions, modelo

from sklearn.metrics import roc_curve
InteractiveShell.ast_node_interactivity = 'last_expr'

palette_roc = ['#006d2c', '#31a354', '#74c476']

def plot_roc_curve(prediction):
    """
    Función que nos devuelve los valores de FPR y TPR
    para representar las curvas ROC
    :param prediction: Spark Dataframe con las predicciones
    del modelo
    :return: fpr, tpr
    """
    results = prediction.select(['probability', 'result'])
    results_collect = results.collect()
    results_list = [(float(i[0][0]), 1-float(i[1])) for i in results_collect]
    fpr = dict()
    tpr = dict()
    y_test = [i[1] for i in results_list]
    y_score = [i[0] for i in results_list]
    fpr, tpr, _ = roc_curve(y_test, y_score)
    return fpr, tpr


def representacion_curvas(dataframe):
    """
    Función que representa un gráfico con las curvas ROC
    de los modelos que se pasan como parámetro mediante
    un Panda Dataframe
    :param: dataframe: Panda Dataframe
    """
    for index, row in dataframe.iterrows():
        fpr, tpr = plot_roc_curve(row['Predictions'])
        plt.plot(fpr, tpr, label='%s (AUC = %0.3f)' % (row['Modelo'] ,row['AUC']),
                color=palette_roc[index])
    plt.plot([0, 1], [0, 1], '--', color='gray')
    plt.xlim([0.0, 1.0])
    plt.ylim([0.0, 1.05])
    ax.set_ylabel('TPR', color=palette[1])
    ax.set_xlabel('FPR', color=palette[1])
    ax.yaxis.set_label_coords(-0.06,0.5)
    ax.xaxis.set_label_coords(0.5, -0.1)
    ax.legend(loc="lower right", fontsize=12)   
    
    
def get_roc_points(dataframe):    
    """
    Función que nos devuelve un Panda Dataframe
    con los puntos de la ROC para exportarlos
    posteriormente y representarlos en Tableau
    :param: dataframe: Panda Dataframe
    :return: roc_pd
    """
    roc_pd = pd.DataFrame()
    for index, row in dataframe.iterrows():
        fpr, tpr = plot_roc_curve(row['Predictions'])
        i = len(tpr)
        temp_pd = pd.DataFrame({'Modelo': np.full(i, row['Modelo']),
                                'FPR': pd.Series(fpr),
                                'TPR': pd.Series(tpr)})
        roc_pd =  roc_pd.append(temp_pd)
    return roc_pd
